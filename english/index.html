Mobile Money Integration/ SMART IVR / PUK CODE REQUEST API

The integration project of Mobile Money services, SMART IVR, and PUK CODE REQUEST API for the partner required the establishment of a secure environment to ensure a reliable and protected connection between the client's servers and ours. Here are the key steps of this process:

Creation of a Certificate Signing Request (CSR):
The first step involved generating a Certificate Signing Request (CSR) from our server. This CSR contains information about our organization and server, necessary to obtain an SSL certificate.
Sending the CSR to the client for signing:
The generated CSR was then sent to the client for them to sign it with their own certificates. This ensures that our communications will be recognized as secure by their system.
Receipt of root and intermediate certificates:
Once the CSR was signed, we received the root and intermediate certificates. These certificates are essential to establish a chain of trust between our servers and those of the partner.
Installation of root and intermediate certificates in the store:
We then installed the root and intermediate certificates in our server's certificate store. This installation allows our server to recognize and trust the certificates issued by the client.
Connectivity testing between servers and certificate verification:
After installing the certificates, we conducted connectivity tests between our server and the client's server. These tests were aimed at verifying that the connection was properly secured and that the certificates were valid and functional.
Modification of the host file:
Finally, we modified our server's host file to ensure that all requests to the client's services used the new secure connections. This step was crucial to ensure that all communications were conducted via secure channels. These steps enabled us to establish a secure and reliable connection, facilitating the successful integration of Mobile Money services, SMART IVR, and PUK CODE REQUEST API for our partner. This project not only strengthened the security of communications between our servers but also improved the efficiency and reliability of services provided to the partner and their clients.
API Implementation:
The integration of the various APIs required for this project follows a precise and structured flow: SPTRANSFER: Funds transfer from the service provider's account to another enrolled account. DEBIT: Management of payments initiated via USSD, website, or application. GETTRANSACTIONSTATUS: Retrieval of the status of previously initiated payment, transfer, or debit operations. REFUND: Used for refund operations in case of need for reversal. It uses the ECW transaction ID as a reference to initiate the refund. VALIDATEACCOUNTHOLDER: Validation of an account holder. GETACCOUNTHOLDERINFO: Request for user details, returning information such as names, profile, and account status. Each API follows a specific schema for requests and responses, ensuring smooth and secure communication between systems.
Technologies Used
API / .NET / C# / PHP / cURL / PowerShell / OpenSSL.
