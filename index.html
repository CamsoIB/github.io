Intégration Mobile Money / SMART IVR / PUK CODE REQUEST API

Le projet d'intégration des services Mobile Money, SAMRT IVR, et PUK CODE REQUEST API pour le partenaire nécessitait la mise en place d'un environnement sécurisé afin de garantir une connexion fiable et protégée entre les serveurs du client et les nôtres. Voici les étapes clés de ce processus :

Création d’un CSR (Certificate Signing Request) :
La première étape consistait à générer une demande de signature de certificat (CSR) depuis notre serveur. Cette CSR contient des informations sur notre organisation et notre serveur, nécessaires pour obtenir un certificat SSL.
Envoi du CSR au client pour signature :
La CSR générée a ensuite été envoyée au client pour qu'ils puissent la signer avec leurs propres certificats. Cela garantit que nos communications seront reconnues comme sécurisées par leur système.
Réception des certificats racine et intermédiaire :
Une fois la CSR signée, nous avons reçu les certificats racine et intermédiaire. Ces certificats sont essentiels pour établir une chaîne de confiance entre nos serveurs et ceux du partenaire.
Installation des certificats racine et intermédiaire dans le store :
Nous avons ensuite installé les certificats racine et intermédiaire dans le magasin de certificats de notre serveur. Cette installation permet à notre serveur de reconnaître et de faire confiance aux certificats émis par le client.
Test de connectivité entre serveurs et vérification des certificats :
Une fois les certificats installés, nous avons effectué des tests de connectivité entre notre serveur et celui du client. Ces tests avaient pour but de vérifier que la connexion était correctement sécurisée et que les certificats étaient valides et fonctionnels.
Modification du fichier host :
Enfin, nous avons modifié le fichier host de notre serveur pour s'assurer que toutes les demandes vers les services du client utilisaient les nouvelles connexions sécurisées. Cette étape était cruciale pour garantir que toutes les communications se feraient via des canaux sécurisés. Grâce à ces étapes, nous avons pu établir une connexion sécurisée et fiable, permettant l'intégration réussie des services Mobile Money, SAMRT IVR, et PUK CODE REQUEST API pour notre partenaire. Ce projet a non seulement renforcé la sécurité des communications entre nos serveurs, mais a également permis d'améliorer l'efficacité et la fiabilité des services fournis au partenaire et à ses clients.
Implémentation des API :
L'intégration des différentes API nécessaires à ce projet suit un flux précis et structuré : SPTRANSFER : Transfert de fonds du compte du fournisseur de services à un autre compte inscrit. DEBIT : Gestion des paiements déclenchés via USSD, site web ou application. GETTRANSACTIONSTATUS : Obtention du statut des opérations de paiement, de transfert ou de débit précédemment initiées. REFUND : Utilisé pour les opérations de remboursement en cas de besoin de réversion. Il utilise l'ID de transaction ECW comme référence pour initier le remboursement. VALIDATEACCOUNTHOLDER : Validation d'un titulaire de compte. GETACCOUNTHOLDERINFO : Requête des détails de l'utilisateur, retournant des informations telles que les noms, le profil, et l'état du compte. Chaque API suit un schéma précis pour les requêtes et les réponses, assurant une communication fluide et sécurisée entre les systèmes.

API .NET C# PHP cURL PowerShell OpenSSL.

Intégration Call Center

1.1 Exigences
La Société Nationale d'Électricité du Burkina Faso (SONABEL) a mis en place un centre d'appels pour améliorer son service client. Ce projet a été réalisé avec la plateforme Altitude Enghouse Company.
1.2 Scope
Le centre d'appels utilise une interface scriptée qui guide les agents lors des appels entrants. Cette interface permet de : Fournir des informations aux clients. Recueillir les préoccupations des clients. Stocker des informations sur les clients. Générer des tickets pour résoudre les problèmes des clients.
Détails du Projet
2.1 Plateforme Utilisée
Le centre d'appels utilise la solution d'Altitude Enghouse Company, intégrée avec divers scripts pour automatiser et guider les interactions avec les clients.
2.2 Fonctionnalités Clés
Interface Utilisateur pour les Agents : Une interface intuitive et interactive qui guide les agents à travers le processus de communication avec les clients. Information Client : Affiche les informations pertinentes sur le client dès le début de l'appel, permettant à l'agent de mieux comprendre et répondre aux besoins du client. Recueil de Préoccupations : Permet aux agents de documenter les préoccupations et les demandes des clients de manière structurée. Gestion des Tickets : Génère automatiquement des tickets pour chaque problème signalé, assurant un suivi et une résolution efficaces.
2.3 Processus d'Appel
Réception de l'Appel : L'agent reçoit un appel entrant, et l'interface affiche les informations du client et un script pour guider l'agent. Fourniture d'Informations : L'agent suit le script pour fournir les informations demandées par le client. Recueil de Préoccupations : L'agent documente les préoccupations du client dans le système. Création de Ticket : Un ticket est généré pour chaque problème, avec des détails sur la préoccupation du client et les actions nécessaires pour la résoudre.
